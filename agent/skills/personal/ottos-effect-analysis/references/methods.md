## 一、区域层级对比的两种做法

**方法 A：试点区域 vs 上层整体（首选）**

上层整体的变化代表大环境（季节性、促销、市场波动），试点区域的变化减去它就是净效果。上层数据天然可得，不用额外筛对比区域。

| 试点范围 | 大环境基准（上层整体） |
|---------|-------------------|
| 某些片区 | 所在营运区整体 |
| 某些营运区 | 所在省区整体 |
| 某些省区 | 所在大区整体 |
| 某个大区 | 全国整体 |

`策略净效果 ≈ 试点区域变化 − 上层整体变化`。例：广州片区动销率同比 +5%，广东省区 +1%，净效果约 +4pp。

**方法 B：试点区域 vs 同级别对比区域（补充）**

事后找一个同级别、条件相似但没上策略的区域做对照。本质是"事后准 AB"。

⚠️ **选择偏差**：试点区域往往本就是条件更好或更积极的区域，事后对比容易高估/低估效果。用此法须在报告里写明对比区域的选择依据和局限。

## 二、DID 匹配对照门店

给每家试点门店从非试点池里找一家特征最相似的门店做对照，按"前后变化之差"算净效果。粒度最细、最接近 AB。

**匹配维度**（按优先级，确保策略外条件尽量一致）：

| 维度 | 说明 |
|------|------|
| 业务基线 | 上线前 3 个月核心指标均值——**最关键，决定两组是否真可比** |
| 资质特征 | 医保资质（有无 88 类阴凉区）、品类结构（中药/参茸/OTC 占比）|
| 基础特征 | 门店面积、城市等级、营运区 |
| 环境特征 | 商圈类型、临近门店密度 |

**匹配规则**：优先匹配基线接近的；Top 1 作对照，1:1，对照门店不复用；至少 30 对，少于此样本不足。

**DID 计算**：

| 时期 | 试点均值 | 对照均值 |
|------|---------|---------|
| 上线前 | A | C |
| 上线后 | B | D |

`净效果 = (B−A) − (D−C)`——同时扣除了门店本身差异（A vs C）和大环境趋势（D−C）。计算和显著性交给 [did_analysis.py](#四did_analysispy-用法)，不用手算。

## 三、平行趋势检验（DID 的前提）

DID 可信依赖**平行趋势假设**：不做策略的话，试点和对照本应有相似的变化趋势。靠两道检验保证，缺一不可：

1. **基线水平检验**（看一个点）：两组上线前的指标基线差异应接近 0。
2. **平行趋势检验**（看一段斜率，更关键、易漏）：拉上线前 **6 个月以上**历史，看两组曲线斜率是否平行。

> 基线一致 ≠ 趋势平行。可能两组某个时点水平相同，但试点一直在涨、对照在跌——基线检验会通过，DID 却会把试点"本来就有的上升势头"误算成策略效果。这跟 AB 的 AA 验证同源，区别是 AA 看"当下水平"（一个点），平行趋势看"历史斜率"（一段线）；事后归因没有事前分组，所以这一关 AA 替代不了。

趋势不平行：这对匹配不可用，重选对照；普遍不平行说明该场景不适合 DID，改用同比+区域对比并下调可信度。`did_analysis.py` 会基于上线前历史自动给出平行趋势结论。

## 四、did_analysis.py 用法

```bash
python3 scripts/did_analysis.py input.json          # 人类可读摘要
python3 scripts/did_analysis.py input.json --json   # 结构化 JSON，可喂给 build_report.py
```

输入 JSON：

```json
{
  "metric": "米效", "unit": "元/米/天", "baseline_period": "上线前3个月", "metric_type": "amount", "alpha": 0.05,
  "pairs": [
    {"pair_id": 1, "t_pre": 85, "t_post": 92, "c_pre": 86, "c_post": 88}
  ],
  "parallel_trend": {
    "periods": ["M-6","M-5","M-4","M-3","M-2","M-1"],
    "treatment_mean": [80,81,83,84,85,85],
    "control_mean": [81,82,83,85,86,86]
  }
}
```

- `metric_type`：`amount`（金额/数量型，如米效、销售额——额外报相对基线 %）或 `rate`（比率/百分点型，如动销率、毛利率——只报绝对 pp、不报相对 %，数据按百分点数值填，如 42 而非 0.42）。默认 amount。
- `pairs`：每对一行，`t_`=试点门店，`c_`=对照门店，`pre/post`=上线前/后均值。
- `parallel_trend`（可选但强烈建议）：上线前各期两组均值，用于平行趋势检验；不给则脚本提示下调可信度。

输出含：净效果（绝对值 + 按基线 %）、配对 t 检验 p 值与 95% 置信区间、单侧符号检验（同向比例，小样本更稳健，如 5/5 → p≈0.03）、平行趋势结论。

**门店对数少（约 < 30，尤其 < 10）**：t 检验难显著，不强求显著性。主看"同向比例 + 净效果幅度"，把结论定位为方向性证据而非确证，并在报告标注可信度（可信度分级见 SKILL.md 正文）。

## 五、净增量法（增量来源拆解 / Source of Volume）

当策略改变的是**"哪些对象被覆盖"**——上/下架一批商品、扩大或调整覆盖的员工、增减试点门店——覆盖对象的产出变化不等于策略的真实贡献，里面混着策略**减量部分**带来的损失。净增量法按对象的"新增/持续/退出"三态把效果拆开，扣掉减量的减益，得真实净效果。商品场景行业称 **Source of Volume / Incrementality 分析**，是评估新品、线扩展、品类调整的既定方法；同样的"增/留/减"拆法可同构用于员工、门店等任何"覆盖对象集合发生增删"的策略。

**先认准"对象"是什么**——对象 = 策略作用的最小个体，"产出"按对象类型取对应指标：

| 覆盖对象 | 典型策略 | 增量（新覆盖） | 存量（持续覆盖） | 减量（退出覆盖） | 产出指标 |
|---------|---------|----------------|------------------|------------------|---------|
| 商品 SKU | 组货、选品、品项优化、促销换品 | 新上架商品的销售 | 保留商品的销售变化 | 淘汰商品的原销售 | 销售额 / 毛利 |
| 员工 | 培训、工具、激励的覆盖范围调整 | 新覆盖员工的提效 | 原覆盖员工的提效变化 | 退出覆盖员工的原产出 | 人效 / 人均产出 |
| 门店 | 某运营动作的试点增减 | 新纳入门店的提升 | 原试点门店的提升变化 | 退出门店的原产出 | 店效 / 店均销售 |

通式：`净增量 = 增益 − 减益`（完整口径还需计入"持续覆盖对象"的连带变化，见口径一）。

**按策略动作分两种口径**，先认准策略到底动了什么：

### 口径一·增删类策略（覆盖的对象集合发生增删）

凡是"覆盖的对象集合变了"的策略都属此类：新增一批被覆盖对象、移出一批旧对象，持续覆盖的对象也会被连带影响（商品=腾出的排面/调整的陈列，员工=分流的工作量，门店=分流的客流）。标准做法是按对象**状态**把产出变化拆成三块（商品场景业内称销售桥 / assortment decomposition）：

- **毛增量（增益）** = 新增对象（本期新覆盖、上期未覆盖）的产出
- **保留变化** = 两期都覆盖的对象，覆盖后 − 覆盖前的产出变化（±）
- **减益** = 退出对象（本期移出覆盖、上期有）覆盖前的产出

`净增量 = 毛增量 + 保留变化 − 减益`

| 对象状态 | 取数 | 归类 |
|---------|------|------|
| 新增对象（本次新覆盖，如新上架商品 / 新覆盖员工 / 新纳入门店） | 覆盖后产出 | 毛增量（+） |
| 持续对象（两期都覆盖） | 覆盖后 − 覆盖前 | 保留变化（±） |
| 退出对象（本次移出覆盖，如淘汰商品 / 退出员工 / 退出门店） | 覆盖前产出 | 减益（−） |
| —— | 三项加总 | 净增量 |

**这三项加总恰好等于"所在对象总体的净变化"**（一个品类的全部 SKU、一个运营区的全体员工、一个区域的全体门店）——口径一是把总体拆成三块来源看清楚，口径二是直接看总体，两者闭合一致、不冲突。

**关键认知·产出转移（商品场景称需求转移 demand transference）**：退出对象损失的产出，不等于它原来的产出。移出一个对象，它的产出有一部分会**转移**给持续/新增的同类对象被接住——删掉一个 SKU，需求转移到其他商品；缩减一个员工的覆盖，他的工作量转移到其他员工；撤掉一家试点门店，客流转移到周边门店。只有真正流失出总体的那部分（商品场景称 walk rate）才是净损失。三态拆解会自动处理转移：被新增接住的算进毛增量、被持续对象接住的算进保留变化、真正流失的才落在净额里——三项加总不重复计、也不漏计。

> 反例：只算"新增 − 退出"、漏掉保留变化，就会把转移到持续对象的产出当成净损失，**低估净增量**。补上保留变化这第三项，结果就等于对象总体、无偏差。

例：某次商品调整（组货 / 促销选品 / 品项优化）后，某品类店均日销从 1000 → 1080 元。拆开看：新增商品 +120（毛增量），保留商品 +0，淘汰商品原日销 40（减益）。净增量 = 120 + 0 − 40 = 80 = 品类整体的 +80。若淘汰的需求转移到了保留商品（保留 +25、新增仍 120、淘汰 40），净增量 = 120 + 25 − 40 = 105 = 品类整体的 +105。无论需求怎么转移，三项加总永远等于品类整体净变化。员工、门店同构：某提效工具的覆盖从 100 人扩到 150 人，新覆盖 50 人的提效（毛增量）+ 原 100 人的提效变化（保留）− 退出覆盖者的原产出（减益）= 该群体总人效的净变化。

#### 精度分层·按汇报对象选一层

净增量里有个绕不开的**交叉转移**：新增对象的产出里混着从退出/持续对象转来的（虚高了），退出对象全额扣减但部分产出其实被别的对象接住了（虚低了）——两边方向相反、量级相近。要多精确，选一层即可，不必一上来就追求最细：

| 层级 | 怎么算 | 适用 |
|------|--------|------|
| **① 粗算（默认）** | `新增 − 退出`（保留变化能拿到就一并加）。默认"虚高 ≈ 虚低"自动对冲，不单独拆交叉转移 | 绝大多数业务汇报，够用 |
| **② 半精确（关键汇报）** | 算法同①，但在结论里加一句注明转移方向与对冲假设（话术见下），既诚实又不显粗糙 | 给吕总/陈总、A 级项目材料 |
| **③ 精确（上系统）** | 估每个退出对象的流向，得真实流失率（walk rate）与留存率，逐对象归因。商品场景有成熟模型（市场篮 market basket、需求转移 demand transference）；员工/门店要精确需用各自的产出流向分析，逻辑同构、工具不同 | 只在做算法模型时（商品的组货/推荐/定价等）；人工汇报用不上 |

**② 的固定注释话术**（可直接复制进报告）：

> 新增对象产出中含部分由退出 / 持续对象转移而来，退出损失中亦含部分被其他对象接住，两者方向相反、量级相近，本次粗算视为对冲；若需精确需做交叉转移（商品场景即 demand transference）分析。

> 为什么粗算够用：要的是"策略整体净增量"这个总数时，虚高与虚低在总额里本就相互抵消（三项加总 = 对象总体净变化是会计恒等）；只有当你要追问"某个新增对象自身贡献了多少真增量"时，才需要③逐对象拆转移。

### 口径二·调结构不增删（对象集合不变，只调内部权重/资源）

对象集合不变，只调内部的权重 / 位置 / 价格 / 资源分配。被倾斜对象涨的部分，可能是从同类其他对象搬来的内部蚕食——商品调主推搬走同品类其他商品的销售，门店做资源倾斜搬走其他门店的客流，员工激励倾斜搬走其他人的产出。

| 口径层 | 看什么 | 归类 |
|-------|-------|------|
| 被倾斜对象（主推/调价/换位/资源倾斜） | 上线前后产出变化 | 毛增量 |
| 同类·未倾斜对象 | 是否被搬走（下滑） | 减益（蚕食，负） |
| 所在对象总体 | 是否真增 | 净增量 = 毛增量 − 蚕食 |

判定：被倾斜对象大涨 + 同类明显下滑 + 对象总体≈持平 → 多为内部搬运，净增量≈0；对象总体同步增长 → 真净增量。

**易错点**：

1. **先认准口径**。增删类（对象集合变了：组货、品项优化、员工覆盖增减、门店试点增减）是口径一，扣退出对象减益；调结构类（对象不变只调权重：主推、陈列、价格、资源倾斜）是口径二，扣同类蚕食。用错口径会算偏：增删类若套口径二只盯总体，会漏掉"退出对象本可保留的产出"这笔账；调结构类若套口径一找不到被退出的对象，无从下手。增删类同时也挤压持续对象时，两口径可叠加——先用口径一算增删净额，再用口径二看对象总体确认新增不是从持续同类搬来的。

2. **三块齐了才精确等于对象总体；只算两头是粗算近似**。算满新增、持续、退出三块，加总就精确等于对象总体净变化（会计恒等）。实操常只算"新增 − 退出"两头、省掉保留变化，这是靠交叉转移对冲的粗算近似（见上"精度分层"①），多数汇报够用；但要心里有数：漏掉保留变化时，若退出对象的产出大量转移到持续对象，会低估净增量——拿不准就把保留变化补上。

3. **盘子边界决定算不算准**。"同类范围"太窄会漏掉真正被搬走或被接住的对象、高估净增量；太宽会混入与策略无关的波动、噪声盖过信号。边界按对象类型定（商品=同品类/同货架，员工=同岗位/同组，门店=同区域/同层级），须事先定清写进口径。

4. **与对照方法正交、不互相替代**。同比/区域/DID 扣的是**外部环境**（大盘、季节），净增量法扣的是策略**内部的减量**。理想链路：先用前者扣大环境得到"策略净效果"，再用净增量法确认这个效果不是被退出对象损失 / 内部搬运抵消掉的虚增。两者叠加，不是二选一。

5. **双口径并看**（商品场景常用销量 + 毛利）。策略若汰换低毛利单品，销量口径可能显示净增量≈0（被淘汰损失抵消），毛利口径却净增 > 0（结构向高毛利迁移）。两口径都算，避免漏掉结构优化的价值。员工、门店同理可设双口径，如员工的产出量 + 产出质量、门店的销售 + 利润。

> 行业依据（均为**商品 / assortment 场景**的成熟方法；推广到员工、门店属同构应用，不另引来源）：Clarkston Consulting 把销量拆为 Base + Incremental − Cannibalization，明确"蚕食抬高 gross 增量但降低 net benefit"；NielsenIQ 把 assortment 层面的 incrementality 定义为"真正新增、而非在商品间重新分配"的销售，并把 delist（删减商品）作为腾挪增量的手段；删减商品的需求转移业内称 **demand transference**，RELEX、Oracle Retail 用它预测淘汰商品的销售有多少转移到保留商品（retained / 被接住）、多少真正流失（walk rate）——RELEX 指出删掉商品的真实损失是它的"独特增量销售"部分、而非当前总销售额；Ipsos、Decision Analyst 将 Source of Volume / Cannibalization 列为评估线扩展的既定方法。
