# 线下零售 AB 实验设计

> 来源：数智中台部 · 数据运营组。本 Skill 整合行业方法论与团队实践，仅限集团内部学习使用，请勿外发。

为线下零售场景设计合适的 AB 实验方案。与互联网 AB 实验的主要区别：**线下零售适用于按门店/地区/商品分组，而非按用户分组**。

## 核心概念

### 分组维度

| 分组维度 | 适用场景 | 注意事项 |
|---------|---------|---------|
| **门店** | 组货策略、陈列方案、定价策略 | 最常用；需按城市等级/面积/商圈类型做分层匹配 |
| **地区** | 区域性策略试点 | 样本量少，需要更长观测期 |
| **商品** | 品项优化、促销策略 | 注意商品间的替代效应 |

### 溢出效应

实验组的处理影响到了对照组，导致对比不公平。典型场景：实验组门店调整了选品，附近对照组门店的顾客跨店购买，对照组指标被人为抬高。

**关键业务经验**：按需处理即可。线下零售的门店选址一般会错开、避免内部竞争，溢出影响通常有限；只有同商圈内自家门店扎堆时才需把它们分到同一组。

### 替代效应（Cannibalization）

优化部分商品/品类时，可能挤占关联品类的货架或销售。局部指标提升但整体可能没有改善。与溢出效应是不同的问题：溢出效应靠**分组设计**解决，替代效应靠**指标监控**发现。

### AA 实验

AB 之前的必要关卡：两组先都用旧方案跑一段，验证分组质量。两组都没动方案，数据本就该差不多——若差异明显，说明分组没分好，重新分组再跑，通过了才进入 AB。

## 执行步骤

### 输出前硬规则

AB 方案先过 6 条硬规则，避免方案看似完整但无法执行：

| 规则 | 要求 |
|------|------|
| 分组单位先定 | 先判断门店 / 地区 / 商品，不能默认按用户分组 |
| 核心指标只有 1 个 | 实验成败只绑定一个主指标，辅助指标用于解释 |
| AA 必须出现 | 每份方案都要有 AA 时长、验证指标、通过标准、不通过处理 |
| 护栏可触发动作 | 护栏必须连接停止条件或暂停排查动作 |
| 决策规则前置 | 实验开始前写清推广、优化、放弃、分层推广的判定 |
| 样本量降噪优先 | 门店数有限时先匹配 / 分层 / ANCOVA / CUPED 降方差，再考虑延长周期 |

### 场景判别

先按策略对象选择实验设计：

| 策略对象 | 推荐分组 | 关键设计点 |
|---------|----------|------------|
| 门店策略 | 门店分组，优先分层匹配 | 城市等级、面积、商圈、基线指标均衡 |
| 区域策略 | 地区分组 | 样本少，周期更长，需标注方向性证据风险 |
| 商品 / 品类策略 | 商品或门店-商品组合分组 | 必须监控替代效应和关联品类 |
| 少量试点门店 | 匹配对设计 | 一店一对照，事后用配对分析或 DID 降噪 |
| 可开关策略 | Switchback 可选 | 同一门店不同时间段对照，注意周期和残留效应 |

### 第一步：收集实验信息

向用户确认以下信息（如用户未提供，主动询问）：

**必填信息**：

| 信息 | 说明 |
|------|------|
| 策略/功能名称 | 要测什么 |
| 当前方案 | 对照组用什么方案 |
| 新方案 | 实验组的变更内容 |
| 分组对象 | 按什么维度分组（门店/地区/商品） |
| 可用样本量 | 有多少门店/商品可参与实验 |
| 核心指标 | 主要看什么指标来判断新方案更好 |
| 业务预期 | 预期提升幅度 |
| 样本差异信息 | 门店/商品之间有哪些差异维度（城市等级、面积、品类等） |
| 时间约束 | 有没有必须在某日期前出结论的要求 |

> **业务预期的单位，跟着核心指标的类型走**——一个常见错误是给数值型指标硬套 pp，写之前先判断核心指标是哪一类：
> - 指标本身就是**占比/比率**（动销率、转化率、执行合规率、缺货率、毛利率等）→ 提升用 **pp（百分点）**。例：动销率从 65% 提到 68%，写「≥3pp」。
> - 指标本身是**数值/金额**（米效、坪效、客单价、销售额、连带率等）→ 提升用 **%（相对）或绝对值**，**不要写 pp**（米效不是百分比，给它套 pp 没有意义）。例：米效预期提升写「≥3%」或「≥1.5 元/米」。
>
> 如果用户给的预期单位和指标类型不匹配（如「米效 ≥2pp」），按上面的规则纠正过来，并在方案里说明换算口径。

**可选信息**（用于统计设计，如用户不提供则按经验估算）：

| 信息 | 说明 | 默认处理 |
|------|------|---------|
| 核心指标基线值 | 当前指标平均水平 | 向用户确认或基于行业经验估算 |
| 最小可检测效应 MDE | 提升多少才值得推广 | 使用用户填写的"业务预期"作为 MDE |
| 门店间标准差 | 指标在门店间的波动 | 按基线 × 变异系数估：比率/强度型（米效、坪效、动销率、转化率等）8-12%，销售额绝对值类 15-20%（详见 2.9） |

### 第二步：设计实验方案

按以下结构输出完整方案：

#### 2.1 实验假设

AB 实验的假设是固定的——**新方案的核心指标优于旧方案**（对照组为旧方案基准），这本就是做实验的目的。每次只需把具体的核心指标和预期提升幅度填进去（取自第一步的"核心指标"和"业务预期"），不必重写统计学的零假设/备择假设。

> 例：新陈列方案能让试点门店的销售米效高于现有陈列，预期提升 ≥ 3%。

#### 2.2 分组策略

**分层维度**：

说明按哪些维度做分层，确保实验组和对照组在关键维度上分布一致：

| 分层维度 | 分层方式 | 理由 |
|---------|---------|------|
| （如：销售等级） | （如：月销 <8万 小店／8-20万 中型店／20-50万 大店） | 销售规模直接决定基线水平，是常见的分层维度 |
| （如：城市等级） | （如：一线/新一线/二线/三线及以下） | 不同层级基线指标差异大 |
| （如：门店面积） | （如：<100㎡/100-150㎡/>150㎡） | 面积影响SKU数和坪效 |

**分组操作方式**（根据门店差异程度和实验范围选择）：

**方式一：分层内随机分配**
在每个分层内随机分配到实验组/对照组，比例建议 1:1。适用于同层门店差异较小的场景。

**方式二：匹配设计 — 推荐**

当门店间差异大、简单随机难以通过 AA 验证时，使用匹配设计。根据实验范围选择具体方法：

**匹配组设计（Matched Group Design）**— 区域内所有门店都参与实验时使用：
1. 将该区域所有门店按核心指标（如动销率）从高到低排序
2. 按顺序单双数分组：第 1 名进实验组、第 2 名进对照组、第 3 名进实验组……
3. 两组在核心指标上的基线高度一致，大幅提升 AA 通过率

**匹配对设计（Matched Pairs Design）**— 只有少量门店试点时使用：
1. 为每个试点门店，在未试点门店中找一个条件最相似的门店（面积、商圈类型、客流、基线指标等维度匹配）作为对照
2. 形成一一对应的"试点-对照"门店对
3. 适用于试点门店数量少、无法做大规模分组的场景（如自动陈列试点 5 家店，配对 5 家对照店）

**溢出效应处理**（按需）：线下零售的门店选址一般会错开，避免内部竞争，溢出影响通常有限。只有同商圈内自家门店扎堆时，才把它们分到同一组，避免实验组调整后顾客跨店购买、抬高对照组。

#### 2.3 AA 实验设计

| 项目 | 内容 |
|------|------|
| AA 时长 | 建议 1-2 周（至少覆盖 1 个完整业务周期） |
| 验证指标 | 核心指标 + 主要辅助指标的基线值 |
| 通过标准 | 两组核心指标基本一致，差异在正常波动范围内（没有明显差距） |
| 不通过处理 | 重新分组或调整分层条件，重跑 AA |
| 额外检查 | 两组在各分层维度上的分布是否均衡 |

#### 2.4 观测指标

| 指标类型 | 说明 | 设计要求 |
|---------|------|---------|
| **核心指标** | 判断实验成败的主要依据 | 仅 1 个，与实验假设直接对应 |
| **辅助指标** | 补充理解实验效果的来源和机制 | 1-3 个，帮助解读核心指标变化的原因 |
| **护栏指标** | 底线监控，触发停止条件 | 设定明确的告警阈值 |

> **替代效应提醒**：当策略仅优化部分商品或品类时，必须将「关联品类/未优化品类的销售变化」设为护栏指标，防止局部优化导致整体受损。

#### 2.5 实验周期

| 阶段 | 时长 | 内容 |
|------|------|------|
| AA 期 | X 周 | 验证分组质量 |
| AB 期 | Y 周 | 正式实验（至少覆盖 2-3 个完整业务周期） |
| 分析期 | Z 周 | 数据清洗、统计检验、结论输出 |

说明周期设计的考虑因素：补货周期、促销节奏、季节性等。

**业务节点冲突检查（线下零售必做）**：

实验周期默认是连续的几周，但中间很可能撞上大促、换季、月底盘点这些业务节点。撞上之后实验组和对照组的数据都会被剧烈扰动，真实的策略效果（可能就几个百分点）会被节点带来的波动盖掉。

排期前先把实验窗口跟下面这些节点比对，有冲突就处理：

| 业务节点 | 影响 | 处理方式 |
|---------|------|---------|
| 大促（618 / 双 11 / 周年庆等） | 客流和客单价剧烈波动，掩盖策略效果 | 实验窗口主动错开；错不开就把大促期数据单独标记、分析时剔除 |
| 季节切换（换季、流感季、过敏季） | 品类需求结构变化 | 确认两组在季节敏感品类上分布一致；必要时延长周期跨过切换点 |
| 月底/季末盘点、调价 | 短期销售异常 | 标记异常日，分析时排除 |

如果用户没说明实验期会不会撞上这些节点，主动提醒用户确认。

#### 2.6 风险与应对

根据实验具体场景，识别可能的风险并给出应对方案。常见风险包括：

| 风险 | 说明 |
|------|------|
| 季节性/节假日 | 外部因素干扰实验结果，两组同期对比可消除 |
| 新奇效应 | 短期效果虚高，需要足够长的观测期让效果趋于稳态 |
| 样本流失 | 门店关店/调整导致样本减少，需预留备用 |
| 执行偏差 | 门店未按方案执行，需监控执行合规率 |
| 替代效应 | 优化品类挤压关联品类，需监控整体品类表现 |

#### 2.7 停止条件

| 条件 | 触发阈值 | 动作 |
|------|---------|------|
| 护栏指标告警 | （设定具体阈值） | 暂停实验，排查原因 |
| 严重负向效果 | （如：核心指标连续 X 天低于对照组 Ypp） | 终止实验，回滚 |
| 样本量不足 | 有效样本低于最小要求的 X% | 延长实验期或终止 |

#### 2.8 实验结果与决策映射

这一节规定"实验跑完后，不同结果对应什么业务动作"。**决策规则须在实验开始前与业务方、决策者达成一致**，避免数据出来后再争论"是否算成功、要不要推广"，陷入被动。

| 实验结果 | 业务动作 |
|---------|---------|
| 核心指标提升达到预期幅度、确认是策略带来的（不是随机波动）、护栏未破 | 在所有门店推广 |
| 核心指标有提升但没到预期幅度（涨幅太小不值得推） | 不推广，或优化策略后再测 |
| 核心指标基本没变化 | 不推广、维持原方案；若样本量本就不足，先延长周期/扩大门店范围补足后再测 |
| 核心指标变差 或 护栏指标破线 | 放弃该方案 |
| 整体正向、但部分门店类型变差 | 分层推广：只在效果好的门店类型推，其余不推或再优化 |

> **"预期提升幅度"怎么定**：指提升达到多少才值得全量推广，需结合业务收益与推广成本确定，并非越小越好，应由业务方给出（取决于公司成本模型）。该值在 2.9 统计设计中对应 MDE（最小可检测效应）。

#### 2.9 统计设计（可选，非必做）

> 本节及 `scripts/` 下两个脚本为可选项，仅在需要量化精度时使用。默认方案中样本量只做一句话定性判断，不展开功效计算。统计在 AB 中只回答一件事：两组差异是策略有效还是随机波动。方案的篇幅与结论以业务设计（分组 / 指标 / 周期 / 护栏）为主线。

**分析单位**：按门店分组时每家门店为 1 个样本，核心指标取门店指标值（连续量），两组均值比较用独立样本 t 检验，样本量按门店数算（非顾客数）。匹配对设计改用配对 t 检验，方差更小、更省样本。

**固定参数**（业界默认）：α = 0.05（误报风险 ≤ 5%）；统计功效 1-β = 0.80。

**样本量判断**（默认只写一句话定性结论，不附公式与参数对照表）：

- 心算：每组所需门店 ≈ 16 × (σ / MDE)²（σ、MDE 同单位）。MDE 取业务预期提升幅度；σ 取门店间标准差，无历史数据时按「基线 × 变异系数」估——比率 / 强度型（米效、坪效、动销率、转化率）8-12%，销售额绝对值类 15-20%。
- 比较所需门店数与可用门店数（总数 ÷ 2），给出「充足 / 偏紧 / 不足」。

**样本不足时的应对**（线下常态，按杠杆强弱排，不必都做）：

1. 降方差：匹配对 / ANCOVA / CUPED，用基线扣除门店间结构差异——门店数有限时最有效的杠杆。
2. 扩大参与门店范围。
3. 与业务方约定把可推广门槛定在现有样本能稳健检测的水平。
4. 延长 AB 周期：主要作用是覆盖完整业务周期、消化新奇效应（降偏差），对降低门店间方差作用有限，非补样本手段。

> **门店数过少（约 < 10 家）**：经典两组 t 检验难以达到显著，不追求显著性。改用配对 + DiD + ANCOVA 降噪，判定看「同向比例 + 提升幅度」（如 5 对全部同向，符号检验 p ≈ 0.03），按事前约定的决策规则拍板。策略可开关时用 switchback（时间段作对照），无对照组时用合成对照。必要时定位为方向性试点 + 上线监控，而非确认性实验。

**实验后结论判断规则**（对应业务动作见 2.8）：

| p 值 vs α | 效应量 vs MDE | 结论 |
|-----------|--------------|------|
| p < 0.05 | ≥ MDE | 策略有效，达到推广门槛 |
| p < 0.05 | < MDE | 提升真实但幅度不足，按 2.8 评估 |
| p ≥ 0.05 | — | 证据不足；若样本未达所需，多为未检出而非无效，补样本后复测 |

**脚本**（填业务参数即可；结论只落一句话，不要把脚本多行输出搬入文档）：

- 事前估样本量：`python3 scripts/sample_size.py --baseline <基线> --mde <预期提升> --available <可用门店总数>`（σ 用变异系数估加 `--cv`，相对 MDE 用 `--mde-rel`，降方差加 `--rho`）
- 事后判结论：`python3 scripts/analyze_result.py --treatment <实验组> --control <对照组> --mde <推广门槛>`（匹配对加 `--paired`；传 `--required-per-group` 可区分真无效与样本不足）

### 第三步：输出自检清单

| 检查项 | 通过条件 |
|--------|---------|
| 分组考虑混淆因素 | 不是简单随机，而是按关键维度做了分层或匹配 |
| 包含 AA 验证 | 有明确的 AA 时长、指标和通过标准 |
| 样本量有评估 | 已判断可用门店数是否够支撑实验（结合门店差异和预期提升幅度，或至少说明是否充足） |
| 周期覆盖业务周期 | 至少 2-3 个完整的业务周期（补货/促销/结算） |
| 业务节点无冲突 | 实验窗口已跟大促/换季/盘点比对，有冲突的已主动错开或标记剔除 |
| 有停止条件 | 明确什么情况下提前终止实验 |
| 决策规则已约定 | 实验结果对应的业务动作（推广/优化/放弃）已事前与业务方商量好 |
| 溢出效应已评估 | 已判断是否存在自家门店扎堆，如有则同组处理 |
| 替代效应已考虑 | 如仅优化部分品类，关联品类已纳入护栏指标 |

### 第四步：提示下游衔接

> 本节是**提示用户后续步骤**用的：方案设计完成后，在输出末尾简短提及即可，**不要自动调用其他 skill**。后续工具由用户按需另行发起。

AB 实验是策略闭环里"验方案"这一环，跟前后两个工具共用指标口径：

- **上游（定指标）**：如果还没设计指标体系，先用 `ottos-north-star-metric` 定北极星和护栏，再拿过来直接作本实验的核心指标和护栏指标，不要重新定一套。
- **下游（看效果）**：实验设计里定的核心指标、护栏、分组方式，实验结束后用 `ottos-effect-analysis` 复盘时直接复用——而且因为有了对照组，效果分析会简单很多（实验组 vs 对照组直接对比，不用费劲重建可比性）。
- **反过来**：如果某个策略已经全量上线、没做 AB 来不及补，那就走 `ottos-effect-analysis` 的"无 AB"路径做事后归因。

## 输出格式

**输出方式**：默认在当前项目目录生成一份 Markdown 文档，同时把内容在聊天窗口同步输出。Markdown 文档作为正式产品文档，用于策略立项、实验方案确认后归档进项目资料；聊天窗口同步展示，方便用户多轮对话修改。文件默认命名 `[策略名称]-AB实验方案.md`。

**文档头**（文档开头固定加）：

| 项 | 值 |
|----|----|
| 策略名称 | - |
| 日期 | 当天（YYYY-MM-DD） |
| 版本 | v0.x 草稿 / v1.0 初稿 / v2.0 正式 |

> 版本怎么定：方案里只要还有**未与业务方确认的项**（基线/MDE 待定、样本量待核、决策规则未事前约定等），就标 **v0.x 草稿**；团队内部评审过、口径齐了才到 v1.0 初稿；干系人签字确认才 v2.0 正式。AI 首次生成、没经过人确认的方案，默认就是 v0.x 草稿。

**固定输出结构**（写进文档的正式内容，每次保持一致；下游衔接只在对话提、不入文档）：

1. 一句话结论（实验验证什么、怎么判定成败——聚焦业务设计，不写成"功效不足/样本不够"这类统计结论）
2. 实验假设
3. 分组策略
4. AA 实验设计
5. 观测指标（主 / 辅助 / 护栏）
6. 实验周期（含业务节点冲突检查）
7. 风险与应对
8. 停止条件
9. 实验结果与决策映射
10. 统计设计（可选，非必做；默认只需一句话判断样本够不够，不展开完整功效分析，除非用户要深入）
11. 自检清单及通过状态

**其他**：

- 关键参数一律用表格呈现，便于评审和执行
- 百分点用 `pp`、百分比用 `%`，不要混用；样本量、MDE 等写明取值或估算依据

**文体**：输出的是正式实验方案文档，用书面陈述。章节标题用名词短语，不用动词短语或对话口吻（如「分组策略」不写成「怎么分组」）；正文客观陈述，避免「请你确认 / 你要…」这类对话口吻。

## 注意事项

- 样本量是配角：门店数有限是线下常态，不作为关键瓶颈或首要风险呈现。正式方案中只写一句话定性结论（充足 / 偏紧 / 不足 + 应对），不展开功效分析，不附所需样本量或不同 ρ 的对照表。
- 样本不足的应对按杠杆强弱：优先降方差（匹配对 / ANCOVA）与扩门店，延长周期主要降偏差而非补样本（详见 2.9）。门店数过少时经典 t 检验难显著，按 2.9「门店数过少」一节处理。
- 辛普森悖论：分组务必考虑混淆变量，避免分组看都好、合并看反而差的陷阱。
