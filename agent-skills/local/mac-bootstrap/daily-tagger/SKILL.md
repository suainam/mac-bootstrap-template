# 🎯 每日总结自动打标指南 (Daily Tagger)

在生成工作总结时，你需要分析每一条工作内容，并根据以下标准为其附加相应的标签。这些标签将在季度复盘时被 Dataview 自动聚合，用于直接生成《六维度绩效自评》与《成长与提升》报告。

## 1. 绩效维度 (Performance Dimensions)

| 标签 | 核心特征 | 典型示例 |
|---|---|---|
| `#绩效-专业知识` | 解决深度技术或业务域问题；算法建模应用；架构或引擎迁移；模型/规则调优。 | "使用马氏距离+DBSCAN完成ABtest选店" / "解决Trino到Hologres的函数不兼容问题" |
| `#绩效-计划组织` | 并行多项目推进；把控节奏与交付；提前识别并规避风险（如对照店偏差、数据波动）。 | "目录外清理完成试点，跑通完整流程" / "预判了手工更新对数据稳定性的影响，提前同步范围" |
| `#绩效-沟通协作` | 跨部门需求对齐；推动上下游系统对接；解决业务落地卡点；拉通产研协同。 | "跟产研团队对齐了目录外清理的上下游系统对接和数据流" |
| `#绩效-学习发展` | 业务理解从单点扩展到全链路；技术上掌握并落地了新栈（如从纯SQL到机器学习/云平台）。 | "对供应链的理解从单点优化扩展到了全链路" / "掌握并在生产应用了聚类算法" |
| `#绩效-创新` | 业务或技术上的“第一次”；用更科学的方法（算法/自动化）替代传统人工经验。 | "第一次用算法组合替代传统人工选店" / "首次将自有品牌纳入独立测算模型" |
| `#绩效-专业影响力` | 数据直接驱动业务决策；通过硬核指标（如动销率、毛利、库存降低）证明策略有效性；建立科学评估框架。 | "通过ABtest数据说服业务部门扩大试点" / "硬指标证明目录外清理使库存降低32%" |

## 2. 成长维度 (Growth Dimensions)

| 标签 | 核心特征 | 典型示例 |
|---|---|---|
| `#成长-新业绩` | 可量化的硬指标提升、创造了多少流水/毛利、模型大范围上线。 | "模型带动销售额+1.06亿、毛利+6397万" / "Q1动销率提升1.78%" |
| `#成长-新贡献` | 沉淀了可复用的方法论；将临时方案沉淀为标准SOP、自动化工具或日常运营标配。 | "动销分析报表全链路自动化，成为运营标配工具" / "沉淀了ABtest效果评估标准框架" |
| `#成长-新创新` | (偏重结果转化) 通过创新手段直接解决了长期的业务或技术痛点。 | "核心指标迁移至预计算引擎，彻底解决跨引擎查询慢的痛点" |
| `#成长-新成长` | 角色转变（从执行者到闭环负责人）；看问题维度拔高；工程化思维加强。 | "独立闭环了探索、设计、协同、落地的全链路" / "沟通从被动接需求转为主动推协同" |

## 3. 复盘维度 (Review Dimensions)

| 标签 | 核心特征 | 典型示例 |
|---|---|---|
| `#复盘-做得好` | 策略与业务目标高度对齐；主动预判风险；科学用数据说话。 | "组货策略有效转为增量驱动" / "主动规避了对照组选择偏差" |
| `#复盘-需改进` | 进度偏慢；样本量不足；缺乏长效反馈机制；过度依赖上游不稳定数据。 | "报表上线后业务反馈收得不够及时" / "上游手工表未做数据治理影响稳定性" |

---

## 📝 打标规则要求

1. **精准少即是多**：每条总结末尾最多打 **1~3** 个最核心的标签。不要为了打标而打标，如果没有符合的维度，则不打标。
2. **格式规范**：标签直接跟在句子末尾，用空格与正文隔开。例如：`完成新店组货模型线上化，提升全店动销率。 #绩效-专业知识 #成长-新业绩`
3. **结合上下文**：分析 Agent 传递的原始对话记录，如果对话中体现了“推翻了之前的设计”、“跟前端对齐了接口”、“排查了很久的 Bug 终于用新方法解决”，需要敏锐地打上对应的 `#绩效-沟通协作` 或 `#绩效-创新` 等标签。
